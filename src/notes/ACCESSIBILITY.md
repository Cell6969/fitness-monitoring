Accessibility Checklist

- Use semantic HTML (sections, nav, main, header, footer).
- Provide `alt` text for meaningful images; decorative images may use empty alt.
- Ensure interactive controls are keyboard-focusable and have visible focus styles.
- Use sufficient color contrast for text and important UI elements.
- Use accessible labels (aria-label, aria-labelledby) where needed.
- Use `details`/`summary` for FAQ for built-in keyboard behavior.
- Test with screen reader and keyboard-only navigation.
